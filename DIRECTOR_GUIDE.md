# VibeFilming 导演手册

> 给**导演**看的，不是给程序员看的。你不用懂代码，也不用跑命令。
> 这份文档告诉你两件事：**① 这个 AI 手里有哪些"工具"可以用；② 你想改它的拍法时，去改哪个文件。**

---

## 一、你怎么用它

你的入口就是**跟它对话**。直接说人话下需求即可，例如：

- "用九宫格分镜给我做一段 30 秒的武打短片"
- "把刚才第 3 个镜头的剑改成红色，重做这一段"
- "给成片配一段紧张的鼓点 BGM"

它会自己拆解、出图、出片、自我审查、不满意自己重做，满意了才交给你。**你不需要告诉它用哪个工具**——下面这张工具表是给你"知道它能干什么"用的，不是让你去点的。

---

## 二、它手里的工具（人话版）

### 🎬 开项目 / 管项目
| 工具 | 干嘛的 |
|---|---|
| `project_create` | 开一个新片子的工作区，定下预算（最多出多少段视频） |
| `project_open` | 切回之前某个片子继续做 |

> 想看片子进行到哪、花了多少预算？用通用 `file_read` 读 `projects/<pid>/manifest.json` 即可，不再有专门工具。

### 📋 分镜（开拍前的剧本）
分镜**不再是独立工具**——用通用 `file_write` 把分镜 JSON 写到 `projects/<pid>/storyboard.json`（几个镜头、每镜多长、谁出场、什么机位、做什么动作），用 `file_read` 读回。写法/必含字段见 `skills/skill_movie/SKILL.md`。

### 🎭 角色 / 道具 / 场景（保证前后长得一样）
没有专门的"角色档案"工具——一个角色/道具/场景**就是几张图**：

1. 用 `gen_image` 把每个角色/道具/场景的形象图先出好（正面/侧面/背面、或道具的某个状态）；
2. 自己记住「主体名 → 图 url」的对应关系；
3. 出视频时把这些 url 放进 `gen_video_t2v` 的 `reference_images`，所有镜头都拿同一批图当参照。

> 这是**防"变脸"**的关键：先把每个角色/道具的样子用 gen_image 定死并过审，后面所有镜头都喂同一批参考图，人物就不会拍着拍着换了张脸。具体出图模板/防漂移方法见 `skills/skill_movie/SKILL.md`。

### 🖼️ 出图 / 出视频
| 工具 | 干嘛的 |
|---|---|
| `gen_image` | 文字生成一张图，或在已有图上做修改（出关键帧、海报、九宫格都用它） |
| `gen_video_t2v` | **生成视频**的唯一入口。可以喂参考图（reference_images）/上一段视频来保证连贯。提交后立刻返回，不当场等 |
| `query_video_task` | 查视频出好了没（会自动等待 + 显示进度条），出好就存盘 |
| `cancel_video_task` | 取消还没开始跑的视频任务（已经在跑的取消不了，会照常计费） |

### ✂️ 后期剪辑
| 工具 | 干嘛的 |
|---|---|
| `video_concat` | 把多段视频**硬拼**成一条（最快，要求各段格式一致） |
| `video_crossfade` | 两段之间做**交叉溶解**转场（会慢一点） |
| `video_trim` | 剪掉视频的某一段（保留 [开始,结束] 区间） |
| `video_speed` | 快放 / 慢动作（注意：不许用慢放来凑时长，那是作弊） |
| `video_overlay` | 画中画（小窗叠在大画面角落） |
| `video_fade` | 开头/结尾加黑场淡入淡出 |
| `video_portrait` | 横屏转竖屏（居中裁剪，不变形） |
| `burn_subtitle` | 在画面上烧录字幕文字（永久叠上去） |

### 🎵 声音 / 配乐
| 工具 | 干嘛的 |
|---|---|
| `gen_audio_bgm` | 生成一段纯音乐 BGM（需要配密钥） |
| `query_audio_task` | 查 BGM 出好了没 |
| `audio_amix` | 把 BGM 混进视频原声里（音量一般压低到不盖对白） |
| `tts` | 文字转语音/配音（需要配密钥） |

> ⚠️ 规矩：**BGM 一律最后单独做**，不让视频模型自己带 BGM（那样多段拼起来音乐会断裂）。

### 👁️ 审片（AI 自己当导演看一眼）
| 工具 | 干嘛的 |
|---|---|
| `vlm_understand` | 让 AI **看**一张图或一段视频回答问题：有没有穿帮、人物一不一致、对不对得上诉求。出片前的质检全靠它 |
| `extract_frames` | 从视频里抽几张静帧（用来 debug 或做缩略图） |

### 🧰 通用工具（不只影视，干啥都能用）
`code_run`（跑代码）/ `file_read`（读文件）/ `file_write`（写文件）/ `file_patch`（改文件局部）/ `web_scan`（看网页）/ `web_execute_js`（操作网页）/ `update_working_checkpoint`（记工作笔记防遗忘）/ `ask_user`（卡住了问你）/ `start_long_term_update`（长任务结束后沉淀经验）

## 三、想改它的"拍法"，去改哪里

这个 AI 的行为分**两层**，你想改什么，对号入座改对应的文件——**改完不用重启、不用动代码，下次对话自动生效**。

### 第 1 层：技能（skills/）——"某件事具体怎么做"

这是你**最常改**的地方。每个技能是一个文件夹，在 `skills/` 下：

| 你想改… | 改这个文件 |
|---|---|
| 分镜怎么排、角色怎么不变脸、多段怎么链式承接、审片问什么、BGM 怎么做、串行还是并行 | `skills/skill_movie/SKILL.md`（影视制作主流程已整合进这一个文件） |
| 写画面 prompt 的句式、禁忌词 | `skills/skill_prompt_engineering/SKILL.md` |

**加一个全新技能**：在 `skills/` 下新建一个文件夹 `skill_你的名字/`，里面放一个 `SKILL.md`（对齐 Anthropic Agent Skills 标准）：
- 文件顶部用 `---` 包一段 YAML：`name`（小写连字符，如 `skill-你的名字`）+ `description`（一句话写清"这个技能干什么 + 什么时候该用"，这段会自动出现在 AI 的技能菜单里驱动触发）
- `---` 下面写详细做法正文

建好就自动生效，AI 下次开机就知道多了这个技能。**不用改任何代码。**（脚本、参考资料、素材可选放进同目录的 `scripts/` `references/` `assets/`，没有就不用建。）

### 第 2 层：底线规则（system prompt）——"什么绝对不许干"

文件：`assets/sys_prompt_film.txt`。这里是 AI **不可违反的底线**，只有 8 条，比如：

- 多镜头开拍前必须先定分镜，定了几段就交几段，不许偷偷砍
- 出片前角色形象必须先做好并审过
- 每个产物出来必须审一遍，没过审的绝对不许拼进成片或交付

> **改这层要谨慎**：这里写的是"底线"，不是"具体怎么做"。
> - 想改**怎么做**（用什么手法、出几张图）→ 改第 1 层 skill。
> - 想改**底线本身**（比如"允许不审片直接交"）→ 才动这层。
> 经验法则：底线只说"必须达到什么状态"，**不要在这里写死某个具体工具名或格式**——那样会把手脚绑死，回头换拍法就互相打架。

## 四、举个例子：我想让它默认都用九宫格

1. 它**已经有**这个技能了 → 关键帧并行法已并入 `skills/skill_movie/SKILL.md`，你直接说"用九宫格/关键帧并行做"它就会读。
2. 想让它**默认**（不用每次都说）→ 改 `skills/skill_movie/SKILL.md`，加一句"多镜头默认走九宫格并行，只有武打/长镜头连续运动的片子才改用链式承接"。
3. 不用碰 system，不用碰代码，下次对话生效。

> 注：九宫格适合"镜头之间是切换"的片子（广告/MV/产品）。**武打、追逐这种讲究连续动作的，反而该走链式承接**——这条判断就写在上面第 2 步那个文件里。
