# Skill · Prompt Engineering（调用前最后 30 秒自检）

> **何时读我**：每次正式生成素材前。
> **我只管"这一次生成要求是否忠实继承上游方案"**；这一段戏怎么拆镜头、用哪些参考、节奏怎么排，是上游编导/故事板环节已经定好的事，本文件不重新规划，只负责调用前自检。

## 调用前自检 · 命中任意一条就停手改

| 死罪 | 为什么会翻 | 正确做法 |
|---|---|---|
| **跳过参考素材** | 人物、道具、场景或分镜只写在文字里，模型会重新脑补 | 已定资产必须作为真实参考参与生成；文字只负责说明用途 |
| **故事板没用起来** | 分镜画了但没有成为视频构图和节拍依据 | 生成前确认本段故事板已经作为构图蓝图被使用 |
| **凭空编外形** | 临时补的发型、服饰、五官会和参考图打架 | 角色外形只来自拍摄表和终版参考图，没写的别编 |
| **主体指代漂移** | 多人物时反复写"他/她/老人/女孩"，模型容易串脸或生成重复人物 | 先给每个主体稳定标签，后续动作、对白、位置都用同一标签 |
| **对白丢失或概述化** | “他说了几句”无法生成准确口型和情绪 | 逐字台词、说话人、音色和情绪必须完整保留 |
| **声音意图矛盾** | 有口型但没台词、有配乐但没说明、不想要配乐却没禁用 | 声音计划必须明确：对白、环境声、配乐、静默分别是什么 |
| **用文字修参考图缺陷** | 参考图本身错，后续会继续污染 | 回到素材环节修参考图，不靠文字反向补救 |

## 一、先消费故事板/拍摄表，别自由发挥

如果当前项目已经产出了结构化拍摄表（如全片计划、片段表、镜头节拍、音频计划、故事板），那么最终生成要求**不能自由改写，必须按拍摄表逐项编译**。这是调用前第一道闸门。

逐项消费规则：
- **镜头节拍**：按顺序展开，不跳节拍、不把关键节拍合并成一句概述。
- **主体定义**：多主体镜头先定义每个主体的稳定标签和对应参考来源；后续每次写动作、表情、对白、站位都用同一标签，少用"他/她/那个人"这种可被误绑定的代词。
- **对白/声音**：逐字抽取台词，写明说话人、音色、情绪和说话时机；同一角色每段都重复完整音色，不写"音色同上"。
- **画面内文字**（屏幕/纸条/手机/日历）：每句逐字写出，并安排成清楚可读的镜头，不要写"出现一行行文字""弹出提示"。
- **关键信息/动作破绽**：每条对应一个明确镜头或特写，不写"发现异常""感觉不对"。
- **音效、配乐和旁白**：写具体声音事件和音乐意图，不写"气氛紧张"这种无法执行的氛围词。没有角色对白的剧情片，也要有清楚的声音叙事方案；除非用户明确要求纯音乐或无声，否则用旁白、画外音、环境声和音乐承担信息推进。
- **故事板/分镜图**：它是故事流动和镜头节拍蓝图，必须真实参与本段生成；不能只在文字里说"参考故事板"。故事板必须是多宫格/多面板的手绘草图（线稿/弱色彩）表达，能看出镜头顺序、主体走位、动作变化、情绪或信息递进；复杂动态、强反转、多关键信息点的戏，不许只靠一张静态氛围图硬生。故事板的手绘弱色彩是有意为之——颜色和外观由资产参考决定，故事板只锁故事流动和构图，所以别把故事板的线稿风格当成成片画风继承。
- **故事板元素必须有资产支撑**：故事板只决定故事怎么推进、镜头怎么排、元素在画面中如何出现；角色、场景、关键道具的具体外观、材质和风格必须来自终版资产参考。生成本段视频前确认：故事板里出现的角色、场景和关键道具，是否都有对应资产参考一起参与；缺哪个就回素材环节补，别让模型仅凭故事板外观脑补。
- **分清故事板和画风锚点**：带故事板做视频参考时，故事板只控制构图、顺序、走位、动作方向和节奏；画面风格、材质、光影、细节由角色/场景/道具参考和全片风格锚点决定。最终生成要求必须把这两类参考的职责分清，避免把故事板的视觉质感误当成成片画风。
- **参考素材取舍**：只使用本镜头真正需要的角色、场景、道具、构图或声音参考。参考越多越容易互相抢权重；不相关素材会制造身份、风格和空间冲突。

编译完做一次字符串级核对：拍摄表里每句对白、每句画面内文字，是否逐字进入生成要求；本段故事板和终版参考资产是否真的被使用。缺一项就不许调用。

## 二、Seedance 特殊符号（这是 prompt 语法，不能删）

Seedance 会靠特殊符号识别音频和文字意图。这些不是工具参数，而是生成要求本身的一部分：

| 内容 | 符号 | 规则 |
|---|---|---|
| 音乐 / BGM | `（）` | 需要配乐时写清音乐情绪、强弱、节奏和对白避让；不需要配乐时明确写无背景音乐。 |
| 音效 / 环境声 | `<>` | 写具体声源和动作声，不写"气氛紧张"这类抽象氛围词。 |
| 台词 / 实声对白 | `{}` | 有人物开口、口型、说话、喊叫、低语等语义时，必须写逐字台词；只写"他说/她嘴唇微动"不够。 |
| 旁白 / 画外音 / 内心 OS | `{}` | 需要生成声音时也必须写逐字文本，并说明它是画外音/内心 OS、人物不张嘴或不查口型。 |
| 字幕 / 标题 / 屏幕叠字 | `【】` | 只有任务明确需要模型直接生成画面文字时使用；不用字幕就不要写。画面内道具文字仍要逐字写清。 |

硬规则：出现说话语义但没有 `{逐字台词}`，就会变成嘴动无声或含混噪声；需要声音但没写对应符号，模型很可能不会按预期生成。

使用边界：
- `{}` 只放真正要生成出来的人声文本。`{无对白}`、`{沉默}` 这类不是台词，会误导模型；无对白应写成普通约束：无人物说话、无旁白、无口型，只保留指定环境声或静默。
- 有对白时，不只写台词本身，还要按拍摄表写清说话时机、语速/情绪、停顿和结尾余韵；需要被打断、急促说完、慢动作留白都可以，但必须明确写成创作意图，不能让模型自己猜。
- 段尾有对白时，生成要求里要说明最后一句结束后是否留反应镜头、呼吸、停顿或环境声收束，避免尾音贴着段边界。

## 三、生成要求怎么写

- 先明确任务类型：新生成、基于参考改写、延展已有内容、局部替换。不要把几种意图混成一句含糊描述。
- 多主体场景必须给每个主体稳定标签，后续每次动作、对白、位置变化都指向具体主体，避免串脸和错人。
- 描述角色外形只允许来自两个来源：拍摄表已写明的稳定特征、终版参考图实际呈现。两者都没说的细节不要临时脑补。
- 多张参考图同时参与时，要在文字里说清每张各自负责什么，别让模型平均地把所有参考的画风都揉进来。典型分工：角色/道具/场景参考负责"长什么样"（外形、材质、配色、身份特征），风格锚图负责"整体调性/光影/质感"，故事板负责"故事流动、构图与镜头编排"。成片画风以资产参考和风格锚图为准，不继承故事板自身的视觉风格。
- **画面细节要写满写活，别只给骨架。** 在不跑偏主体和镜头设计的前提下，把人物、物体和场景写得更具体、更有质感，别用一个笼统类别名带过。物体要写清外形结构、材质质感、表面状态、磨损或反光关系；人脸要写清五官状态、皮肤质感、眼神、细微表情、毛发与光线关系；场景要写清材质、光影层次、空气感和必要的环境动态。这些细节必须服务这一镜的情绪和信息，不要堆砌与剧情无关的杂物，也不要细到喧宾夺主盖过主体或破坏镜头语言。
- **情绪和动作要按镜头内的时间推进分段写，别给一个静止的状态词。** 单写"哭""害怕""愤怒"不够，要写清这一镜里表情、身体和动作如何从起点变化到落点。尤其是固定机位、靠表演撑住的单镜，更要写清情绪曲线：前段是什么状态，中段发生什么变化，后段落到什么表情或动作。动作也一样，写清起势、过程、落点，而不是一个凝固姿势。节奏用镜头内的阶段关系表达，不用精确秒点替代导演节拍。
- 动态描述按镜头顺序写：镜头与视角、主体动作、空间变化、声音。可以写"镜头 1/镜头 2/镜头 3"这种时序分镜，但不要用过细时间戳替代镜头节拍；模型对精确秒点不稳定，节奏靠镜头和动作承接表达。
- 镜头语言要写进生成要求，不能丢。编导定的景别/机位视角/运镜方向和速度/焦点景深（前景背景虚实、焦点是否转移）/光照方向与性质必须逐项编译进 prompt，并用文字暗示首尾镜（opening frame / ending frame）。不要把一段戏写成全程一个固定景别、一种平移；编导设计的镜头变化和递进要照搬，不能在生成要求里被抹平成"拍一下人物做动作"。
- **编导设计的变速（慢镜头/升格、加速/降格）要编译进生成要求。** plan 里写了变速的镜头，就要写清哪个动作或哪个节拍使用慢动作、加速或回到正常速度，以及这样处理服务什么情绪或信息。变速是表达手段，不写清就容易被默认成普通速度，关键瞬间会被一带而过。
- 动作要具体到可见身体部位、速度、幅度和惯性承接。少写"非常悲伤/很紧张"，改成低头、肩膀微颤、手指攥紧衣角、频繁看表、呼吸急促、眼神闪躲这类可拍细节。
- 优先写低缓、连续、可追踪的小动作；除非剧情需要，少把单段写成狂奔、大跳、剧烈翻滚等高爆发动作。高动态动作要拆清楚起势、过程和落点。
- 台词语言统一，专有名词除外；同一角色的音色和说话习惯保持一致。
- 有人物开口就必须有逐字台词并使用台词符号；不需要台词时，把动作改成纯表情或动作，避免“嘴动无声”。
- 段间衔接要写进生成要求：开头承接上一段的情绪/动作/声音/未完成信息，结尾把下一段需要接住的情绪、目标或悬念留出来。不要只写本段孤立画面。
- 文字保持聚焦，别把完整剧本、审查记录和无关设定全塞进去。信息太多会分散模型注意力；无关背景放回上游计划，最终生成要求只保留本镜头必须执行的内容。
- 配乐是导演决策：需要就写清情绪、强弱和对白避让；不需要就明确禁用。不要让模型随机决定。

## 四、不要写工具层内容

- 不在 skill 里规定具体工具名、字段名、参数默认值、文件路径写法、任务轮询方式或底层滤镜。
- 不把工具已经承载的参考输入，在文本里反复复述成“参考某某文件”；文本只说明每个参考素材的叙事用途。
- 不写固定模板。特殊符号是模型可识别的 prompt 语法，必须按需使用；但不要把整段生成要求做成僵硬模板。
- 不用一段通用样板套所有镜头；短片、广告、口播、空镜、动作戏的文本密度和声音意图应随内容变化。

## 五、最后放行条件

正式调用前逐项确认：

- 逐字对白没有丢。
- 画面内文字没有丢。
- 关键动作和反应没有被概述掉。
- 本段故事板已经真实参与生成。
- 本段涉及的角色、场景、道具参考已经真实参与生成。
- 音频意图没有自相矛盾。
- 没有临时新增人物外形设定。
- 没有把工具层字段、参数、路径、模板写进 skill 规则。
