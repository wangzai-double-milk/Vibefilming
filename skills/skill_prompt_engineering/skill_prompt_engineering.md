## 调用前 7 步 checklist（按顺序过）

```
1. 任务类型 3 选 1   →   决定句式
2. 主体定义合规     →   <主体N> + 2-3 静态特征 + 多素材绑定
3. 动态描述顺序     →   镜头 → 主体 → 空间 → 音频
4. 4 类符号必用     →   <音效> {台词} 【字幕】 （音乐🚫）

5. BGM 双闸门 ⭐    →   generate_audio=False（闸门1）+ prompt 末尾禁令句（闸门2）
6. 时长比例双保险   →   "时长：N秒，比例：H:W"（中文"秒"）
```

## 1. 任务类型（写 prompt 必先选一类）

| 类型 | 适用 | 推荐句式 |
|---|---|---|
| **参考 Reference** | 从素材提取元素生成全新视频（动作迁移 / 主体复用 / 氛围借鉴） | `参考<图片N>中的<主体N>，生成...` / `参考<视频N>中的[动作/运镜场景/风格/音效]，生成...` / `参考<音频N>中的[音色]，生成...` |
| **编辑 Edit** | 在原视频基础上局部/全局修改 | `严格编辑<视频N>，将其中的[原特征]修改为[新特征]...` |
| **延长 Extend** | 时间维度延续原视频 | `延长<视频N>，生成...` / `向前延长<视频N>，生成...` |
| **组合任务** | 参考一个 + 编辑另一个 | `参考<图片/视频N>的[参考维度]，严格编辑<视频X>，[具体编辑内容]` |

⚠️ **编辑/延长任务里提到视频时**，**不要**用"参考视频N"——直接用 `<视频N>`，否则会被识别成参考任务。

---

## 2. 主体定义（多角色场景必写）

**句式**：`将<图片/视频N>中的[主体核心特征]定义为<主体N>`

**核心特征要求**：2-3 个**清晰、稳定的静态特征**（服饰 / 发型 / 外观 / 类别），确保唯一可识别性。

**多素材同一主体**（统一绑定）：
```
将<图片1>中的[...]、<图片2>中的[...] 定义为<主体1>
```

**多主体场景**（用稳定标签区分）：
```
将<视频1>中的高个子男人定义为<主体1>(警察)
将另一个矮个子男人定义为<主体2>(小偷)
```

⚠️ **每次涉及主体时需明确指代，避免省略**（每次出现主体，就提及一次）；
⚠️ **使用 Asset 库时**，依然用图1/视频1指代主体——模型不会自动把 Asset 和参考内容对应上，**不能直接用 Asset ID 替代**；
⚠️ 描述简洁、避免冗余、避免语义冲突；
⚠️ 空间关系优先通过参考图表达，少用复杂文字描述。

---

## 3. 动态描述顺序（必须按此顺序）

```
镜头 → 主体 → 空间 → 音频
```

具体为：
1. **运镜 / 镜头切换**（如"镜头一：中景缓推"）
2. **主体动作与表情**（细化到肢体——手、腿、头部 + 程度——幅度、速度、力度，例："缓慢抬手"/"快速转头"/"用力蹬地"）
3. **位置或空间变化**
4. **音频信息**（4 类符号见下）

**时序表达**：
- 按事件发生顺序描述（先主后次）
- ✅ 支持"镜头1 / 镜头2 / 镜头3"分段
- ❌ "0-3 秒"这种精确时间支持不稳定，不要用

**语言规范**：台词语言**必须统一**——中文就全中文，英文就全英文，避免中英混用（专有名词除外）。

---

## 4. 4 类特殊字符（写 prompt 必遵守）

Seedance 靠括号识别音频意图——**不写括号 = 模型自己猜，靠运气**。

| 内容 | 符号 | 例子 | 本项目用法 |
|---|---|---|---|
| **音乐 / BGM** | `（）` | `（背景中播放着快节奏的摇滚乐）` | 🚫 **本项目禁用**——BGM 走后期 `audio_amix` |
| **音效** | `<>` | `<远处传来狗叫声>` | ✅ 主力，丰富现场感 |
| **台词** | `{}` | `{你好，世界}`；小语种需标注：`用日语说道{こんにちは}` | ✅ 主力，对白必用 |
| **字幕** | `【】` | `【第一章：启程】` | ⚠️ 一般禁用（V-2 字幕乱出） |

### ⚠️ 台词陷阱：「张嘴无声」（2026-06-01 公益片实战翻车）

**翻车现场**（[shot_s03_v2](file:///Users/bytedance/Desktop/各种agent的使用尝试/vibefilming-genericagent/projects/p20260601_102739_生成一段30s的公益小视频/shots/shot_s03_v2.mp4)）：
- prompt 写了「老奶奶**夸奖**小女孩 / 邻居**说**真好」**但没用 `{}` 包台词文本**
- `generate_audio=True` 已开
- 翻车结果：老奶奶**张嘴动了但没声音**——模型读到"夸奖/说"语义 → 知道这里"应该说话" → 做出口型 → 但没台词文本可合成 → 哑剧

**根因**：动作语义（夸奖/说/讲/告诉/喊/嘟囔/小声/低语）≠ 台词。Seedance 只接受 `{}` 包裹的**显式文本**作为台词，模糊的动作描述不会自动补全语音。

**3 条铁律**：

| 场景 | 错写法 | 对写法 |
|---|---|---|
| **想要说话** | `老奶奶夸奖小女孩` | `老奶奶笑着 {真棒！你做得真好}` |
| **不想说话但有口型暗示词** | `小女孩开心地告诉奶奶` | `小女孩开心地咧嘴笑`（删掉"告诉"）|
| **想要默剧/无声场景** | `两人激动地讨论` | `两人比划着手势交换眼神，没有说话` |

**触发关键词清单**（写到这些词时自检要不要补 `{}`）：
> 说 / 讲 / 喊 / 叫 / 告诉 / 夸奖 / 表扬 / 道谢 / 招呼 / 嘟囔 / 低语 / 大喊 / 喃喃 / 念叨 / 提醒 / 解释 / 抱怨 / 商量 / 讨论 / 询问 / 回答 / 念 / 唱

> ⚠️ 命中关键词时只有两条路：① 把动作改成"开心地咧嘴笑"这种纯表情描述（去掉口型语义）；② 老老实实补 `{台词}`。**没有第三条路**，否则就是"张嘴无声"翻车。

## 5. ⭐⭐⭐ BGM 显式禁令（治篮球项目 BGM 漏出）

**关键发现**（来自 2026-05-29 篮球片诊断）：
- prompt 里**一个 `（）` 都没写**
- 但视频出来**还是有 BGM**
- 结论：`generate_audio=True` 时 Seedance 会**主动脑补 BGM**，"prompt 不写"不等于"输出无"

### 双闸门铁律（缺一不可）

| 闸门 | 位置 | 作用 | 写法 |
|---|---|---|---|
| **闸门 1（真闸门）** | **工具参数** | 直接关掉 Seedance 的"音轨生成器" | `generate_audio=False` |
| **闸门 2（保险）** | prompt 末尾 | 双保险，告诉模型即便音轨开了也别配乐 | `无背景音乐，仅保留环境音效` |

> 闸门 1 是开关——`generate_audio=true` 等于"请帮我生成完整音轨"，模型一旦被允许出音轨，就会**自己脑补**，prompt 里写"无背景音乐"只是降权重，不能把开关关掉。
> 闸门 2 是文本提示，单独用没用，必须配合闸门 1。

### 默认策略（本项目）

- **大多数场景**：`generate_audio=False` —— 视频纯画面，对白/音效后期 `tts` + `audio_amix` 接进去（音质更可控、不漏 BGM）
- **必须 generate_audio=True 的场景**（极少）：要 Seedance 生成原生唇形对口型对白时——这时**仍然必须在 prompt 末尾追加禁令句**作为闸门 2
- **A-4 MV 模式**：BGM 就是项目本体（外部 BGM 转黑屏视频做参考），完全跳过本节

### prompt 末尾禁令句模板（闸门 2）

```
无背景音乐，仅保留环境音效
```

> ⚠️ 不要照搬 PE 模板里的"...与人物对白"——**当前 shot 没有对白就不要写"对白"**，否则模型可能自己加对白。按"反例扫描"原则：禁令句只列**真要保留的东西**。
>
> - 有对白的镜头：`无背景音乐，仅保留环境音效与人物对白`
> - 无对白的镜头：`无背景音乐，仅保留环境音效`
> - 纯静音镜头：`无任何音频`（同时 `generate_audio=False`）

## 5.5 调用参数 vs prompt：别复述工具已经传过的东西 ⭐

很多 agent 把已经通过工具参数传出的字段又抄到 prompt 里去，有些是"双保险"（OK），有些是"自找麻烦"（NG）：

| 字段 | 工具参数 | 是否在 prompt 复述 | 理由 |
|---|---|---|---|
| `duration` | ✅ 必传 | ✅ **要**复述 `时长：N秒` | 火山官方建议双保险（V-2） |
| `ratio` | ✅ 必传 | ✅ **要**复述 `比例：H:W` | 同上 |
| `generate_audio` | ✅ 工具决定 | ❌ **不要**写 `开启音频` 之类 | 这是开关，不是描述 |
| `reference_entities` | ✅ 工具决定 | ❌ **不要**写"参考已有的角色 X、道具 Y、场景 Z" | 工具会自动注入 ref_image_url，prompt 复述等于多余 |
| `reference_video_url` | ✅ 工具决定 | ❌ **不要**写"参考视频1" | 同上，且编辑/延长任务还会被识别成"参考任务"导致语义错乱 |
| `name` / `task_id` | ✅ 工具决定 | ❌ 完全不该出现 | 这是工程标识，跟模型无关 |

**正例**：直接进入主体定义和动作描述，不要前缀"参考已有的角色 little_boy..."。
**反例**：`参考已有的角色little_boy、道具basketball、场景outdoor_basketball_court生成视频。将参考图中的...` —— 这一整句都是冗余，工具已经把 7 张 ref 图传给 Seedance 了，prompt 直接 `将<图片1>中的...定义为<主体1>...` 即可。

## 6. 时长 / 比例双保险

接口已经传 `duration` / `ratio`，但**官方建议 prompt 最前/最后再复述一遍**，且**用中文"秒"，不能写"s"**：

```

```

## 7. 反例对照扫描表

调用前最后一遍肉眼扫，**命中任意一条 → 改 prompt**：

| ❌ 不要写 | ✅ 改写为 | 原因 |
|---|---|---|
| `（轻快尤克里里 BGM）` | （删掉，加"无背景音乐"） | BGM 走后期 amix |
| `BGM: 摇滚` | （删掉） | 同上 |
| `时长 4s` | `时长：4秒` | 必须中文"秒" |
| `0-3 秒角色抬手` | `镜头一：角色缓慢抬手` | 精确时间不稳定 |
| `参考视频1，编辑视频1...` | `严格编辑<视频1>...` | 编辑任务别带"参考"字样 |
| `Hello {你好}` | `{你好}` 或 `{Hello}` | 中英混用 |
| `生成一个男人打篮球` | `<主体1>(篮球男)在三分线外运球，<主体1>起跳投篮` | 必须每次提及主体 |
| `（古风琴音）<夜风>` | `<指尖拨动琴弦的清音><夜风>` | （）BGM 禁用，改写为环境音效 |
| **`参考已有的角色little_boy、道具basketball、场景outdoor_basketball_court生成视频`** | （直接删掉这句开头） | ⭐ 工具已经注入 ref_image_url，prompt 复述纯属冗余且会跟"任务类型句式"打架 |
| **`...仅保留环境音效与人物对白`（但镜头里没有对白）** | `...仅保留环境音效` | 别照搬模板，按当前 shot 实际有没有对白决定 |
| **shot prompt 里写"无任何白色条纹"修补 entity 缺陷** | 回去重做 `entity_add_view` 把基准图改干净 | 工具自动注入的 ref_image 跟你的反向 prompt 打架，模型乱画 |
| 仅传一张混合人脸+全身的图当人物参考 | 大头照 + 全身照分开传（V-1） | ID 漂移 |
| 多视图 / 三视图当人物参考 | 大头照 + 全身照足够（V-7 双胞胎） | 三视图易触发双胞胎 |

## prompt 模板（按动态描述顺序四件套）

### 单主体单镜头模板

```
<任务类型句式>，
将<图片1>中的[2-3 静态特征]定义为<主体1>。
镜头一：[运镜]。<主体1>[肢体动作 + 程度]。[空间/位置]。<环境音效> {台词}。
无背景音乐，仅保留环境音效与人物对白。
时长：<N>秒，比例：<H:W>
```

### 链式段（≥2 段）模板

```
延长<视频1>。
镜头一：[新运镜]。<主体1>[新动作]。[空间变化]。<新音效> {新台词}。
角色音色 / 服装 / 场景与<视频1>保持一致。
无背景音乐，仅保留环境音效与人物对白。
时长：<N>秒，比例：<H:W>
```

### 多主体对话模板

```
将<图片1>中的[特征]定义为<主体1>(警察)，将<图片2>中的[特征]定义为<主体2>(小偷)。
镜头一：[运镜]。<主体1>[动作]，<主体2>[动作]，二人在[空间]。
<环境音效> 用普通话说道：<主体1>{台词1} <主体2>{台词2}。
不要在同一画面中复制相同人物，不要多人同脸。
无背景音乐，仅保留环境音效与人物对白。
时长：<N>秒，比例：16:9
```

## 何时跳过 PE checklist

- **`entity_add_view` 出基准图**：图编辑模式，prompt 短小直白即可，不需要走 7 步
- **A-4 MV 模式**：BGM 是项目核心，不要写"无背景音乐"禁令
- **debug 抽帧 / 工具调用**：不调模型，无关

其他场景一律走完 7 步再发请求。
